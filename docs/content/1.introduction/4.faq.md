## 安装部署问题

### 无法拉取镜像

- 集群无公网访问能力，需将 VPC 配置到 NAT 网关以确保集群节点可以访问公网）
- 访问 docker.io 受限，使用国内镜像仓库：`veaiops-registry-cn-beijing.cr.volces.com`）


### Helm 安装后域名无法访问

- 确认已安装 Nginx Ingress Controller
- 检查 Ingress 状态和地址，确保域名 DNS 解析指向 Ingress EIP
```bash [Terminal]
> kubectl get ingress veaiops-ingress -n veaiops-system
  NAME              CLASS   HOSTS                   ADDRESS          PORTS   AGE
  veaiops-ingress   nginx   veaiops.example.com     10.x.x.1         80      42m
```
- 确认已申请合法域名如 `veaiops.example.com`，并正确配置 DNS 解析

### 私有镜像仓库认证

使用私有仓库需要配置认证信息（Secret）：

   ```bash [Terminal]
   # 创建 veaiops-system 命名空间
   > kubectl create ns veaiops-system

   # 创建 docker registry 认证 secret
   > kubectl create secret docker-registry regcred \
   --docker-server=YOUR_REGISTRY \
   --docker-username=YOUR_USER \
   --docker-password=YOUR_PASS \
   --docker-email=YOUR_EMAIL \
   -n veaiops-system

   # 覆盖 values.yaml 中的全局配置
   global:
     imageRegistry: "<YOUR_REGISTRY>"
     imagePullSecrets:
       - docker-registry
     security:
       allowInsecureImages: true
   ```

### HTTPS 证书配置

启用 HTTPS 需要配置 TLS 证书，在安装时指定证书 secret 的名称。

```bash
# 创建 TLS Secret
kubectl create secret tls veaiops-tls \
  --cert=fullchain.crt \
  --key=privkey.key \
  -n veaiops-system
```

在 values.yaml 中配置：
```yaml
ingress:
  tlsSecretName: "veaiops-tls"
```


## 使用问题

### LLM 调用失败

- API 限流：调整请求频率或升级配额
- 内容过滤：检查输入是否触发安全策略
- 认证失败：确认 API Key 配置正确

查看详细错误日志进行排查并提交至github issue中寻求帮助。
